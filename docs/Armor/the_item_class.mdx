To have GeckoLib armor, you first need to have an item class.

:::note
This tutorial is specifically for GeckoLib implementation. For information on general-modding armor features, seek
a tutorial or community support.
:::

## Creating the class
<AdditiveCodeHeader />

First, we create the base item class and implement the base constructor:

```java
public class ExampleArmorItem extends Item {
    public ExampleArmorItem(ArmorMaterial material, ArmorType type, Properties properties) {
        super(properties.humanoidArmor(material, type));
    }
}
```

Then, we will implement the `GeoItem` interface and override the `getAnimatableInstanceCache` and `registerControllers` methods:

```java
//highlight-next-line
public class ExampleArmorItem extends Item implements GeoItem {
    public ExampleArmorItem(ArmorMaterial material, ArmorType type, Properties properties) {
        super(properties.humanoidArmor(material, type));
    }

    //highlight-start
    @Override
    public void registerControllers(final AnimatableManager.ControllerRegistrar controllers) {
        // We can fill this in later
    }
    //highlight-end

    //highlight-start
    @Override
    public AnimatableInstanceCache getAnimatableInstanceCache() {
        return null;
    }
    //highlight-end
}
```

Next, let's instantiate an `AnimatableInstanceCache` in a `final` field at the top of our class, and return it in `getAnimatableInstanceCache`:

```java
public class ExampleArmorItem extends Item implements GeoItem {
    //highlight-next-line
    private final AnimatableInstanceCache geoCache = GeckoLibUtil.createInstanceCache(this);

    public ExampleArmorItem(ArmorMaterial material, ArmorType type, Properties properties) {
        super(properties.humanoidArmor(material, type));
    }

    @Override
    public void registerControllers(final AnimatableManager.ControllerRegistrar controllers) {
        // We can fill this in later
    }

    @Override
    public AnimatableInstanceCache getAnimatableInstanceCache() {
        //highlight-next-line
        return this.geoCache;
    }
}
```

And you're done!

For information on adding animation controllers, see the [Animation Controllers](/docs/Concepts/Animation/Controller/overview.mdx) page.

:::info
You might have noticed that this setup is very similar to a GeckoLib item.<br/>
You'd be right! This is exactly the same setup as a GeckoLib item!
:::

## Extra Bits
### I want to use triggered animations on this armor
<Collapsible title="Click to Expand">
    To prepare your armor item class for using triggered animations, you'll need to register the item as a synced animatable.<br/>
    This is done in the constructor, like so:

    ```java
    public class ExampleArmorItem extends Item implements GeoItem {
        private final AnimatableInstanceCache geoCache = GeckoLibUtil.createInstanceCache(this);

        public ExampleArmorItem(ArmorMaterial material, ArmorType type, Properties properties) {
            super(properties.humanoidArmor(material, type));

            //highlight-next-line
            GeoItem.registerSyncedAnimatable(this);
        }

        // ...
    }
    ```

    See the [Triggered Animations](/docs/Miscellaneous/triggerable_animations.mdx) section for more information on triggered animations and how to use them
</Collapsible>
