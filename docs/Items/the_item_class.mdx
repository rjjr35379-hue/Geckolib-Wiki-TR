To have a GeckoLib item, you first need to have an item class.

:::note
This tutorial is specifically for GeckoLib implementation. For information on general-modding block features, seek
a tutorial or community support.
:::

## Creating the class
<AdditiveCodeHeader />

First, we create the base item class and implement the base constructor:

```java
public class ExampleItem extends Item {
    public ExampleItem(Properties properties) {
        super(properties);
    }
}
```

Then, we will implement the `GeoItem` interface and override the `getAnimatableInstanceCache` and `registerControllers` methods:

```java
//highlight-next-line
public class ExampleItem extends Item implements GeoItem {
    public ExampleItem(Properties properties) {
        super(properties);
    }

    //highlight-start
    @Override
    public void registerControllers(final AnimatableManager.ControllerRegistrar controllers) {
        // We can fill this in later
    }
    //highlight-end

    //highlight-start
    @Override
    public AnimatableInstanceCache getAnimatableInstanceCache() {
        return null;
    }
    //highlight-end
}
```

Next, let's instantiate an `AnimatableInstanceCache` in a `final` field at the top of our class, and return it in `getAnimatableInstanceCache`:

```java
public class ExampleItem extends Item implements GeoItem {
    //highlight-next-line
    private final AnimatableInstanceCache geoCache = GeckoLibUtil.createInstanceCache(this);

    public ExampleItem(Properties properties) {
        super(properties);
    }

    @Override
    public void registerControllers(final AnimatableManager.ControllerRegistrar controllers) {
        // We can fill this in later
    }

    //highlight-start
    @Override
    public AnimatableInstanceCache getAnimatableInstanceCache() {
        //highlight-next-line
        return this.geoCache;
    }
}
```

And you're done!

For information on adding animation controllers, see the [Animation Controllers](/docs/Concepts/Animation/Controller/overview.mdx) page.

## Extra Bits
### I want to use triggered animations on this item
<Collapsible title="Click to Expand">
    To prepare your item class for using triggered animations, you'll need to register the item as a synced animatable.<br/>
    This is done in the constructor, like so:

    ```java
    public class ExampleItem extends Item implements GeoItem {
        private final AnimatableInstanceCache geoCache = GeckoLibUtil.createInstanceCache(this);

        public ExampleItem(Properties properties) {
            super(properties);

            //highlight-next-line
            GeoItem.registerSyncedAnimatable(this);
        }

        // ...
    }
    ```

    See the [Triggered Animations](/docs/Miscellaneous/triggerable_animations.mdx) section for more information on triggered animations and how to use them
</Collapsible>

### I want to have my animations only play in my hand, but not in GUI (or vice versa)
<Collapsible title="Click to Expand">
    If you want to handle animations differently depending on the render perspective, you'll need to mark the item as perspective-aware.<br/>
    This is done by adding a method to your item's class, like so:

    ```java
    public class ExampleItem extends Item implements GeoItem {
        private final AnimatableInstanceCache geoCache = GeckoLibUtil.createInstanceCache(this);

        public ExampleItem(Properties properties) {
            super(properties);
        }

        // ...

        //highlight-start
        @Override
        public boolean isPerspectiveAware() {
            return true;
        }
        //highlight-end
    }
    ```

    See the [Perspective-Aware Animations](/docs/Items/perspective_aware_animations.mdx) page for more information on this feature and how to use it.
</Collapsible>
