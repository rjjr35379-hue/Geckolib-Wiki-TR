The first step in making a GeckoLib entity is to create the entity class itself.

:::note
This tutorial is specifically for GeckoLib implementation. For information on general-modding entity features, seek
a tutorial or community support.
:::

## Making the class
<AdditiveCodeHeader />

First, we will make the base entity class:

```java
public class ExampleEntity extends PathfinderMob {
    public ExampleEntity(EntityType<? extends PathfinderMob> entityType, Level level) {
        super(entityType, level);
    }
}
```

Then, we will implement the `GeoEntity` interface and override the `getAnimatableInstanceCache` and `registerControllers` methods:

```java
//highlight-next-line
public class ExampleEntity extends PathfinderMob implements GeoEntity {
    public ExampleEntity(EntityType<? extends PathfinderMob> entityType, Level level) {
        super(entityType, level);
    }

    //highlight-start
    @Override
    public void registerControllers(final AnimatableManager.ControllerRegistrar controllers) {
        // We can fill this in later
    }
    //highlight-end

    //highlight-start
    @Override
    public AnimatableInstanceCache getAnimatableInstanceCache() {
        return null;
    }
    //highlight-end
}
```

Next, let's instantiate an `AnimatableInstanceCache` in a `final` field at the top of our class, and return it in `getAnimatableInstanceCache`:

```java
public class ExampleEntity extends PathfinderMob implements GeoEntity {
    //highlight-next-line
    private final AnimatableInstanceCache geoCache = GeckoLibUtil.createInstanceCache(this);

    public ExampleEntity(EntityType<? extends PathfinderMob> entityType, Level level) {
        super(entityType, level);
    }

    @Override
    public void registerControllers(final AnimatableManager.ControllerRegistrar controllers) {
        // We can fill this in later
    }

    @Override
    public AnimatableInstanceCache getAnimatableInstanceCache() {
        //highlight-next-line
        return this.geoCache;
    }
}
```

And you're done!

For information on adding animation controllers, see the [Animation Controllers](/docs/concepts/animation/controller/overview.mdx) page.

## Registering the entity
Once you have created your entity class, you then need to register it.
This differs depending on the modloader you are using.
